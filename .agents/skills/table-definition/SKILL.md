`src/db/mysql/schema.ts`を正として、`docs/table-definitions/<テーブル物理名>.md`を管理する。

## 作成・更新

1. `docs/database.md`、`docs/table-definitions/_template.md`、MySQLスキーマ、関連マイグレーションを確認する。
2. 1テーブル1ファイルで、テンプレートの見出しと表の列を変更せずに作成・更新する。
3. カラムの型、NULL可否、デフォルト、主キー、制約、インデックス、関連をスキーマと一致させる。
4. 論理名、用途、説明、設計メモはコードとDB設計から確認できる内容だけを記載する。APIだけで検証するルールは記載しない。
5. 該当する制約・インデックス・関連がない場合は「なし」と明記する。
6. スキーマ変更と同じ作業では、定義書も同じコミットに含める。

## レビュー

ユーザーがレビューだけを依頼した場合はファイルを変更しない。

1. MySQLスキーマの全テーブルに対応する定義書があることを確認する。
2. テンプレートの見出しと表の列が維持され、テーブルごとの粒度が揃っていることを確認する。
3. スキーマと定義書のカラム、型、NULL可否、デフォルト、キー、制約、インデックス、関連を照合する。
4. APIだけで検証するルール、根拠のない業務ルール、実装詳細、将来案が記載されていないことを確認する。
5. 問題を重要度順に、対象ファイルと修正内容が分かる形で報告する。
