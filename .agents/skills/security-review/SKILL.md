現在の変更または指定された対象を、APIセキュリティの観点でレビューする。

1. `AGENTS.md`、対象差分、関連する実装・テスト・OpenAPI仕様・Docker・環境変数設定を確認する。
2. OWASP API Security Top 10 2023を基準に確認する。
   - オブジェクト・プロパティ・機能単位の認可
   - 認証とセッション・トークン
   - リソース消費、レート制限、業務フローの悪用
   - SSRFと外部API入力
   - セキュリティ設定、API棚卸し、不要な公開
3. OWASP ASVS 5.0.0とCheat Sheet Seriesを補助基準に確認する。
   - 入力検証、インジェクション、出力・エラー
   - 秘密情報、暗号化、ログ、依存関係
   - CORS、ヘッダー、デバッグ設定、コンテナ設定
4. ユーザー入力からDB・外部通信・ファイル・コマンドまでのデータフローを追う。
5. 依存関係の確認が可能なら`npm audit`を実行し、結果を実装上の到達可能性と分けて扱う。
6. 必要なセキュリティテストが不足している場合は、再現可能なテスト案を示す。

出力は問題を重要度順に先に示し、各指摘に攻撃条件、影響、根拠となるファイルと行、
推奨対応を含める。関連する場合はOWASP API Security Top 10の項目名を付ける。
根拠のない可能性を脆弱性として断定しない。問題がない場合は明記し、未確認事項と
残るリスクを示す。

これは静的・ローカルレビューであり、OWASP準拠認証やペネトレーションテストの代替と
表現しない。外部環境や本番環境への攻撃的な検証は実行しない。
