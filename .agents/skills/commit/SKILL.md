現在のタスクに関係する変更を、独立してレビュー・検証・取り消しできる作業単位ごとにコミットする。

1. `AGENTS.md`を確認する。
2. `git status`、未ステージ差分、ステージ済み差分を確認する。
3. 差分を独立した作業単位へ分ける。実装とそのテスト・OpenAPI・必要なドキュメントは同じ単位として扱う。
4. 未完成、チェック失敗、目的の異なる変更が混在した状態ではコミットしない。
5. 今回コミットする単位に必要なチェックを実行する。最終コミット前は`npm run commit:check`を実行し、現在の変更が原因の失敗を解消する。
6. 対象ファイルだけをステージする。1ファイルに複数目的の変更がある場合は、誤った部分ステージを避けるため分割可能性を慎重に判断する。
7. `git diff --cached`でコミット内容を再確認する。
8. `AGENTS.md`のコミットメッセージ規約に従い、`<type>: <日本語の要約>`形式のメッセージを作成する。スコープが有用な場合は`<type>(<scope>): <日本語の要約>`形式を使用する。
9. typeはファイル種別ではなく変更目的から選び、要約には何が変わるかを具体的に書く。
10. コミットを作成する。依頼範囲の作業が残っている場合は、次の作業単位の実装を継続する。
11. 完了時にコミットハッシュ、メッセージ、対象ファイル、チェック結果を報告する。

調査、レビュー、説明、下書き作成だけの依頼では、ユーザーが明示しない限りコミットしない。
既存コミットのamend、push、Pull Request作成は実行しない。
